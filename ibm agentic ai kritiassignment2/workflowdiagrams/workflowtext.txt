================================================================================
DETAILED NODE FLOW
================================================================================

1. INTENT CLASSIFICATION NODE
   Input: state (contains "query")
   Logic:
     - Analyze query text
     - Match keywords to intents
     - Assign one of: Sales, Technical, Billing, Account, Memory, Unknown
   Output: state["intent"] = determined intent

   Keyword Examples:
     "pricing" / "plan" / "cost" → Sales
     "crash" / "error" / "bug" → Technical
     "refund" / "billing" → Billing
     "password" / "account" → Account
     "previous" / "last issue" → Memory
     Other → Unknown

---

2. MEMORY RECALL NODE (if intent == "Memory")
   Input: state (contains "customer_name")
   Logic:
     - Query SQLite table: conversations
     - WHERE customer_name = input customer_name
     - Retrieve previous interactions
   Output: 
     - state["retrieved_context"] = previous conversations
     - state["retrieved_sources"] = "SQLite Memory"

---

3. RAG RETRIEVAL NODE (if intent != "Memory")
   Input: state (contains "query")
   Logic:
     - Extract keywords from query
     - Match to document keywords:
       • "pricing" → pricing_guide.txt, faq.txt
       • "password" → company_policy.txt, faq.txt
       • "crash" → technical_manual.txt, faq.txt
       • "refund" → company_policy.txt
       • Default → faq.txt
     - Read matched documents (up to k=3)
     - Return as list of dicts: [{"text": content, "source": filename}]
   Output:
     - state["retrieved_context"] = [documents]
     - state["retrieved_sources"] = [filenames]

---

4. ROUTER NODE
   Input: state (contains "intent")
   Logic:
     - Switch statement on intent value
     - Route to appropriate agent
   Routes:
     - "Sales" → sales_node
     - "Technical" → technical_node
     - "Billing" → billing_node
     - "Account" → account_node
     - "Memory" → (already handled, go to approval)
     - "Unknown" → generic_node

---

5. AGENT NODES (Sales, Technical, Billing, Account)
   Input: state (contains "query", "retrieved_context")
   Logic:
     - Read retrieved documents/context
     - Generate appropriate response for query
     - For Billing: Check if "refund"/"cancel" → mark high-risk
     - For Account: Check if "cancel account"/"close" → mark high-risk
     - Generate draft response
   Output:
     - state["draft_response"] = generated response
     - state["department"] = agent name
     - state["approval_required"] = True/False
     - state["risk_type"] = refund/cancel/closure/compensation (if applicable)

---

6. APPROVAL CHECK NODE
   Input: state (contains "approval_required", "risk_type")
   Logic:
     - If approval_required == True:
       → Route to human_node
     - Else:
       → Skip to supervisor_node

---

7. HUMAN APPROVAL NODE (if approval needed)
   Input: state (contains "query", "draft_response", "risk_type")
   Logic:
     - Print to terminal: "Supervisor approval required for {risk_type} request"
     - Read environment variable: HUMAN_APPROVAL_DECISION
     - If demo mode: env var = "yes" (auto-approve)
     - If interactive: Prompt user: "Approve? (yes/no):"
     - Wait for user input
   Output:
     - state["approval_status"] = "approved" or "rejected"
     - If rejected: state["draft_response"] = alternative response

---

8. SUPERVISOR NODE
   Input: state (contains "draft_response")
   Logic:
     - Review response for quality
     - Ensure professional tone
     - Add company branding if needed
     - Verify factual accuracy
     - Finalize response
   Output:
     - state["response"] = final response (ready to send)

---

9. SAVE NODE
   Input: state (full state dict)
   Logic:
     - Extract key fields:
       • customer_name
       • query
       • intent
       • department
       • response
     - Generate timestamp
     - Insert into SQLite conversations table
     - SQL: INSERT INTO conversations VALUES (...)
   Output:
     - Data persisted in memory/memory.db
     - Entry retrievable for future "Memory" queries

---

10. END NODE
    Output: Final response printed to user

================================================================================
EXAMPLE: REFUND REQUEST FLOW
================================================================================

Customer: "I need a refund for my annual subscription."

1. Intent Classification:
   Query matches "refund" keyword → Intent = "Billing"

2. RAG Retrieval:
   "refund" keyword found → Retrieves:
     - company_policy.txt (refund policy)
     - pricing_guide.txt (billing info)

3. Router:
   Intent = "Billing" → Route to Billing Agent

4. Billing Agent:
   Reads: "refund" in query
   Marks: approval_required = True, risk_type = "Refund"
   Generates draft response about refund process

5. Approval Check:
   approval_required = True → Route to Human Node

6. Human Approval:
   Demo mode: HUMAN_APPROVAL_DECISION = "yes"
   Sets: approval_status = "approved"

7. Supervisor:
   Finalizes response: "Your refund has been approved..."

8. Save Node:
   INSERT INTO conversations:
   - customer_name: David
   - query: I need a refund...
   - intent: Billing
   - department: Billing Agent
   - response: Your refund has been approved...

9. Output:
   Final Response shown to customer


================================================================================
EXAMPLE: MEMORY QUERY FLOW
================================================================================

Customer: "What was my previous support issue?"

1. Intent Classification:
   Keywords "previous", "last issue" → Intent = "Memory"

2. Memory Node (NOT RAG):
   Query SQLite: SELECT * FROM conversations WHERE customer_name = "David"
   Found previous entry:
     - Query: "I need a refund..."
     - Response: "Your refund has been approved..."

3. Router:
   Intent = "Memory" → Route to Account/Generic Agent

4. Agent:
   Reads conversation history
   Generates response: "Your previous issue was about a refund request, which was approved."

5. Approval Check:
   Memory queries not high-risk → approval_required = False

6. Supervisor:
   Finalizes response

7. Save Node:
   Save this new query to SQLite

8. Output:
   Response about previous conversation


================================================================================
KEY DECISION POINTS
================================================================================

1. Is Intent Memory?
   → YES: Skip RAG, use SQLite history
   → NO: Use RAG retrieval from docs

2. Is Response High-Risk?
   → YES (refund/cancel/closure/compensation): Get approval
   → NO: Skip approval, go to supervisor

3. Was Approval Given?
   → YES: Proceed with approved response
   → NO: Generate alternative response

4. All Paths Lead To:
   → Supervisor (finalize)
   → Save (persist in SQLite)
   → Output (show to customer)
